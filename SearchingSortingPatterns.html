Linux CLI Tools for Search & Processing

Linux provides powerful command-line tools for searching files, filtering text, and sorting data. These are essential daily tools for any developer or sysadmin — they operate on streams of text and can be composed via pipes (|) to build complex data pipelines without writing a single line of C++.

find — Filesystem Search

find recursively searches the directory tree for files matching given criteria: name pattern, type, size, modification time, permissions, etc.

Syntax: find [path] [expression]
The expression is a combination of tests, actions, and operators evaluated left-to-right.
# Find all .cpp files under current directory find . -name "*.cpp" # Find files modified in the last 7 days find /home/george -mtime -7 -type f # Find files larger than 100 MB find /var/log -size +100M -type f # Find and execute: delete all .o object files find . -name "*.o" -exec rm {} \; # Find files owned by root with SUID bit set (security audit) find / -user root -perm -4000 -type f 2>/dev/null # Find directories named "build" and list their sizes find . -type d -name "build" -exec du -sh {} \; # Case-insensitive search for README files find . -iname "readme*" # Find empty files find . -empty -type f

sort — Sort Lines of Text

sort sorts lines from stdin or files. It supports numeric sorting, reverse order, sorting by specific fields (columns), and uniqueness filtering.

# Sort a file alphabetically sort names.txt # Sort numerically (important! "10" < "9" lexicographically) sort -n scores.txt # Sort in reverse order sort -r -n scores.txt # Sort by 2nd field (column), using comma as delimiter sort -t',' -k2 -n data.csv # Remove duplicate lines (equivalent to sort | uniq) sort -u words.txt # Sort by file size (human-readable) — combine with du du -sh /usr/* | sort -h # Sort processes by CPU usage (field 3) ps aux | sort -k3 -rn | head -10 # Multi-key sort: first by field 2, then by field 1 sort -k2,2 -k1,1 employees.txt

grep, egrep, fgrep — Pattern Search in Text

The grep family searches for lines matching a pattern. They differ in what kind of patterns they accept:

CommandPattern TypeEquivalent toUse When
grepBasic Regular Expressions (BRE)grep -GSimple patterns, BRE syntax
egrepExtended Regular Expressions (ERE)grep -EGroups (), alternation |, +, ?
fgrepFixed strings (no regex)grep -FLiteral string search, fastest
# grep: basic pattern search grep "error" /var/log/syslog grep -i "error" file.txt # case insensitive grep -n "TODO" *.cpp # show line numbers grep -r "malloc" /usr/include/ # recursive grep -v "^#" config.txt # invert: lines NOT matching (skip comments) grep -c "warning" build.log # count matching lines grep -l "main" *.cpp # list filenames that match grep -A3 -B1 "segfault" syslog # 3 lines after, 1 before match # egrep (grep -E): extended regex — no need to escape +, ?, |, () egrep "error|warning|fatal" app.log egrep "^[0-9]{4}-[0-9]{2}-[0-9]{2}" logfile.txt # ISO date at start egrep "(GET|POST|PUT|DELETE) /api" access.log egrep "\b[A-Z]{2,}\b" document.txt # words ALL CAPS # fgrep (grep -F): literal string, no special regex chars interpreted fgrep "std::vector" *.cpp # searches the literal string including <> fgrep -f patterns.txt bigfile.txt # match multiple fixed strings from file # Practical pipeline examples cat /etc/passwd | grep -v "^#" | cut -d: -f1 | sort find . -name "*.log" | xargs grep -l "OOM" ps aux | grep nginx | grep -v grep
Performance tip: fgrep is fastest for literal strings because it uses the Boyer-Moore-Horspool algorithm internally rather than building an NFA/DFA. For searching millions of lines for a fixed string, prefer fgrep over grep.

Binary Search

Binary search finds a target in a sorted array by repeatedly halving the search space. Each comparison eliminates half the remaining candidates, giving \(O(\log n)\) time — searching 1 billion elements requires at most 30 comparisons.

Recurrence:
\[T(n) = T\!\left(\frac{n}{2}\right) + O(1) \implies T(n) = O(\log n)\] At each step: \(\text{mid} = \left\lfloor\frac{\text{lo} + \text{hi}}{2}\right\rfloor\)
— If arr[mid] == target: found.
— If arr[mid] < target: search right half: lo = mid + 1
— If arr[mid] > target: search left half: hi = mid - 1

C++ Implementation

// Binary Search — iterative and recursive versions // Time: O(log n) | Space: O(1) iterative, O(log n) recursive #include <iostream> #include <vector> using namespace std; // Iterative — preferred (no call stack overhead) int binarySearch(const vector<int>& arr, int target) { int lo = 0, hi = (int)arr.size() - 1; while (lo <= hi) { int mid = lo + (hi - lo) / 2; // avoids overflow vs (lo+hi)/2 if (arr[mid] == target) return mid; else if (arr[mid] < target) lo = mid + 1; else hi = mid - 1; } return -1; // not found } // Recursive variant int binarySearchRec(const vector<int>& arr, int target, int lo, int hi) { if (lo > hi) return -1; int mid = lo + (hi - lo) / 2; if (arr[mid] == target) return mid; if (arr[mid] < target) return binarySearchRec(arr, target, mid + 1, hi); return binarySearchRec(arr, target, lo, mid - 1); } // Lower bound: first index where arr[i] >= target (like std::lower_bound) int lowerBound(const vector<int>& arr, int target) { int lo = 0, hi = (int)arr.size(); while (lo < hi) { int mid = lo + (hi - lo) / 2; if (arr[mid] < target) lo = mid + 1; else hi = mid; } return lo; } int main() { vector<int> arr = {2, 7, 13, 19, 24, 31, 37, 45, 52, 60}; int idx = binarySearch(arr, 37); cout << "Found 37 at index: " << idx << endl; // 6 cout << "Not found: " << binarySearch(arr, 100) << endl; // -1 cout << "Lower bound of 25: " << lowerBound(arr, 25) << endl; // 5 }

Bubble Sort — O(n²)

Bubble sort repeatedly passes through the array, comparing adjacent pairs and swapping them if out of order. The largest unsorted element "bubbles up" to its correct position each pass. Simple to implement but impractical for large data.

Complexity Analysis:
Pass 1 makes \(n-1\) comparisons, pass 2 makes \(n-2\), ..., total: \[\sum_{i=1}^{n-1} i = \frac{n(n-1)}{2} = O(n^2)\] Best case (already sorted, with early-exit flag): \(O(n)\)
Worst case (reversed): \(O(n^2)\)  |  Stable: Yes

Visual: Sorting [64, 34, 25, 12, 22, 11, 90]

// Bubble Sort with early-exit optimization // Time: O(n²) worst/avg, O(n) best | Space: O(1) | Stable: Yes #include <iostream> #include <vector> using namespace std; void bubbleSort(vector<int>& arr) { int n = arr.size(); for (int i = 0; i < n - 1; i++) { bool swapped = false; // early-exit optimization for (int j = 0; j < n - 1 - i; j++) { // n-1-i: last i elements are sorted if (arr[j] > arr[j + 1]) { swap(arr[j], arr[j + 1]); swapped = true; } } if (!swapped) break; // no swaps in this pass → already sorted } } int main() { vector<int> arr = {64, 34, 25, 12, 22, 11, 90}; bubbleSort(arr); for (int x : arr) cout << x << " "; // 11 12 22 25 34 64 90 }

Selection Sort — O(n²)

Selection sort divides the array into sorted (left) and unsorted (right) parts. Each pass selects the minimum element from the unsorted portion and swaps it into its correct position. Unlike bubble sort, it makes at most \(n-1\) swaps — useful when write operations are costly.

\[\text{Comparisons} = \sum_{i=0}^{n-2}(n-1-i) = \frac{n(n-1)}{2} = O(n^2)\] Swaps: exactly \(n-1\) — much fewer than bubble sort's \(O(n^2)\) swaps.
Best = Worst = \(O(n^2)\)  |  Stable: No (can skip equal elements)
11, 25, 12, 22, 64] 11 25 12 22 64 Pass 2: find min in [1..4] = 12 @ idx 2 → swap with idx 1 11 25 12 22 64 swap min Passes 3,4: place 22, 25 → Final sorted array: 11 12 22 25 64 Red = swap target position | Orange = minimum found | Green = in final position
// Selection Sort // Time: O(n²) always | Space: O(1) | Stable: No #include <iostream> #include <vector> using namespace std; void selectionSort(vector<int>& arr) { int n = arr.size(); for (int i = 0; i < n - 1; i++) { int minIdx = i; // assume current position has minimum for (int j = i + 1; j < n; j++) { if (arr[j] < arr[minIdx]) { minIdx = j; // found smaller element } } if (minIdx != i) { swap(arr[i], arr[minIdx]); // at most n-1 swaps total } } } int main() { vector<int> arr = {64, 25, 12, 22, 11}; selectionSort(arr); for (int x : arr) cout << x << " "; // 11 12 22 25 64 }

Merge Sort — O(n log n)

Merge sort is a classic divide-and-conquer sorting algorithm. It recursively splits the array in half until subarrays of size 1, then merges adjacent sorted halves. The merge step is the key operation — it takes two sorted arrays and produces a single sorted array in \(O(n)\) time.

Master Theorem: \[T(n) = 2T\!\left(\frac{n}{2}\right) + O(n) \implies T(n) = O(n \log n)\] \(a=2,\ b=2,\ f(n)=n,\ \log_b a = 1\) → Case 2: \(f(n) = \Theta(n^1)\) → \(T(n) = \Theta(n \log n)\)
Space: \(O(n)\) auxiliary for the merge step. Stable: Yes.

Recursion Tree: [38, 27, 43, 3, 9, 82, 10]

// Merge Sort — divide and conquer // Time: O(n log n) always | Space: O(n) | Stable: Yes #include <iostream> #include <vector> using namespace std; // Merge two sorted halves arr[lo..mid] and arr[mid+1..hi] void merge(vector<int>& arr, int lo, int mid, int hi) { vector<int> left(arr.begin() + lo, arr.begin() + mid + 1); vector<int> right(arr.begin() + mid + 1, arr.begin() + hi + 1); int i = 0, j = 0, k = lo; while (i < (int)left.size() && j < (int)right.size()) { if (left[i] <= right[j]) arr[k++] = left[i++]; // <= makes it stable else arr[k++] = right[j++]; } while (i < (int)left.size()) arr[k++] = left[i++]; while (j < (int)right.size()) arr[k++] = right[j++]; } void mergeSort(vector<int>& arr, int lo, int hi) { if (lo >= hi) return; // base case: single element int mid = lo + (hi - lo) / 2; mergeSort(arr, lo, mid); // sort left half mergeSort(arr, mid + 1, hi); // sort right half merge(arr, lo, mid, hi); // merge sorted halves } // Bottom-up iterative merge sort (no recursion stack) void mergeSortIterative(vector<int>& arr) { int n = arr.size(); for (int width = 1; width < n; width *= 2) { for (int lo = 0; lo < n - width; lo += 2 * width) { int mid = lo + width - 1; int hi = min(lo + 2 * width - 1, n - 1); merge(arr, lo, mid, hi); } } } int main() { vector<int> arr = {38, 27, 43, 3, 9, 82, 10}; mergeSort(arr, 0, arr.size() - 1); for (int x : arr) cout << x << " "; // 3 9 10 27 38 43 82 }

Heap Sort — O(n log n)

Heap sort exploits the max-heap property: the parent is always ≥ its children. It first builds a max-heap from the array (heapify), then repeatedly extracts the maximum element (root) and places it at the end. Unlike merge sort, heap sort is in-place with \(O(1)\) extra space.

Two phases:
1. Build max-heap: \(O(n)\) — heapify from \(\lfloor n/2 \rfloor - 1\) down to 0
2. Extract max n−1 times: each sift-down is \(O(\log n)\) → total \(O(n \log n)\)
Stable: No  |  In-place: Yes  |  Worst = Best = \(O(n \log n)\)

Max-Heap: Array [4, 10, 3, 5, 1] → after build-heap

// Heap Sort // Time: O(n log n) always | Space: O(1) | Stable: No #include <iostream> #include <vector> using namespace std; // Sift down: maintain max-heap property for subtree rooted at index i // n = current heap size (shrinks as we extract max elements) void heapify(vector<int>& arr, int n, int i) { int largest = i; // assume root is largest int left = 2 * i + 1; int right = 2 * i + 2; if (left < n && arr[left] > arr[largest]) largest = left; if (right < n && arr[right] > arr[largest]) largest = right; if (largest != i) { swap(arr[i], arr[largest]); heapify(arr, n, largest); // recursively fix affected subtree } } void heapSort(vector<int>& arr) { int n = arr.size(); // Phase 1: Build max-heap — O(n) // Start from last non-leaf: floor(n/2)-1 for (int i = n / 2 - 1; i >= 0; i--) { heapify(arr, n, i); } // Phase 2: Extract max n-1 times — O(n log n) for (int i = n - 1; i > 0; i--) { swap(arr[0], arr[i]); // move current max to sorted end heapify(arr, i, 0); // restore heap with size i } } int main() { vector<int> arr = {64, 34, 25, 12, 22, 11, 90}; heapSort(arr); for (int x : arr) cout << x << " "; // 11 12 22 25 34 64 90 }

KMP — Knuth-Morris-Pratt Pattern Matching

Naive pattern matching checks every possible position in the text, giving \(O(nm)\) time where \(n\) = text length, \(m\) = pattern length. KMP (1977) achieves \(O(n + m)\) by never backtracking in the text. It pre-computes a failure function (also called the partial match table / LPS array) from the pattern, which tells us: if a mismatch occurs after matching \(k\) characters, how far back in the pattern can we jump without re-checking characters we already matched?

LPS Array (Longest Proper Prefix which is also Suffix):
For pattern \(P\), \(\text{lps}[i]\) = length of longest proper prefix of \(P[0..i]\) that is also a suffix.
Example: \(P = \texttt{"ABABCABAB"}\)
\[\text{lps} = [0, 0, 1, 2, 0, 1, 2, 3, 4]\] When mismatch at pattern position \(j\): instead of restarting at \(j=0\), jump to \(j = \text{lps}[j-1]\).

// KMP Pattern Matching // Time: O(n + m) | Space: O(m) for LPS array #include <iostream> #include <vector> #include <string> using namespace std; // Build LPS (Longest Proper Prefix which is also Suffix) array vector<int> buildLPS(const string& pattern) { int m = pattern.size(); vector<int> lps(m, 0); int len = 0; // length of previous longest prefix-suffix int i = 1; while (i < m) { if (pattern[i] == pattern[len]) { lps[i++] = ++len; } else if (len > 0) { len = lps[len - 1]; // try shorter prefix — key insight! } else { lps[i++] = 0; } } return lps; } // KMP search: returns all starting positions of pattern in text vector<int> kmpSearch(const string& text, const string& pattern) { int n = text.size(), m = pattern.size(); vector<int> lps = buildLPS(pattern); vector<int> matches; int i = 0; // index into text int j = 0; // index into pattern while (i < n) { if (text[i] == pattern[j]) { i++; j++; } if (j == m) { matches.push_back(i - j); // match found at position i-m j = lps[j - 1]; // continue searching for more } else if (i < n && text[i] != pattern[j]) { if (j > 0) j = lps[j - 1]; // shift pattern, don't backtrack text else i++; } } return matches; } int main() { string text = "ABABDABACDABABCABAB"; string pattern = "ABABCABAB"; vector<int> lps = buildLPS(pattern); cout << "LPS array: "; for (int v : lps) cout << v << " "; cout << endl; // 0 0 1 2 0 1 2 3 4 vector<int> matches = kmpSearch(text, pattern); if (matches.empty()) { cout << "Pattern not found." << endl; } else { for (int pos : matches) { cout << "Pattern found at index: " << pos << endl; } } // Output: Pattern found at index: 10 }

When to use which algorithm in practice:
  • Binary search — whenever your data is or can be kept sorted (e.g., lookup in a sorted config file, range queries).
  • Bubble/Selection sort — small arrays (<20 elements), teaching, embedded systems where code size matters. Never in production on large data.
  • Merge sort — when stability is required, or when sorting linked lists (doesn't need random access).
  • Heap sort — when you need guaranteed \(O(n \log n)\) with \(O(1)\) extra space, e.g., real-time systems.
  • KMP — single-pattern search in a long text stream; prefer Aho-Corasick for multi-pattern search.
  • std::sort — default choice in C++ for almost all sorting needs.
