Floyd-Warshall Algorithm: All-Pairs Shortest Paths

Floyd-Warshall Algorithm

The Floyd-Warshall algorithm is a dynamic programming algorithm for finding shortest paths in a weighted graph with positive or negative edge weights (but no negative cycles). It finds shortest paths between all pairs of vertices in O(V³) time.

1. Problem Statement

Given a directed or undirected weighted graph with n vertices, find the shortest path between every pair of vertices.

Input

  • Graph with n vertices
  • Weight matrix dist[i][j] = weight of edge from i to j (∞ if no edge)

Output

  • Shortest distance between every pair of vertices
  • Optionally: the actual shortest paths

2. Algorithm Overview

Key Idea

For each intermediate vertex k, check if going through k gives a shorter path from i to j.

Recurrence Relation:

dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])

For all intermediate vertices k from 0 to n-1

Algorithm Steps

  1. Initialize distance matrix with direct edge weights
  2. For each vertex k (0 to n-1) as intermediate:
    • For each pair (i, j):
      • If dist[i][k] + dist[k][j] < dist[i][j], update dist[i][j]

4. Implementation

Complete C++ Implementation

#include <iostream> #include <vector> #include <climits> #include <iomanip> using namespace std; #define INF INT_MAX void floydWarshall(vector<vector<int>>& graph) { int V = graph.size(); // Initialize distance matrix vector<vector<int>> dist(V, vector<int>(V)); for (int i = 0; i < V; i++) { for (int j = 0; j < V; j++) { dist[i][j] = graph[i][j]; } } // Floyd-Warshall algorithm for (int k = 0; k < V; k++) { for (int i = 0; i < V; i++) { for (int j = 0; j < V; j++) { // Avoid overflow if (dist[i][k] != INF && dist[k][j] != INF) { if (dist[i][k] + dist[k][j] < dist[i][j]) { dist[i][j] = dist[i][k] + dist[k][j]; } } } } } // Print shortest distances cout << "Shortest distances between every pair of vertices:" << endl; for (int i = 0; i < V; i++) { for (int j = 0; j < V; j++) { if (dist[i][j] == INF) { cout << setw(5) << "INF"; } else { cout << setw(5) << dist[i][j]; } } cout << endl; } } int main() { // Example graph: 4 vertices int V = 4; vector<vector<int>> graph = { {0, 4, 5, INF}, {INF, 0, INF, 2}, {3, INF, 0, 1}, {INF, INF, INF, 0} }; floydWarshall(graph); return 0; }

5. Path Reconstruction

To reconstruct the actual shortest paths, we maintain a parent/next matrix:

void floydWarshallWithPath(vector<vector<int>>& graph) { int V = graph.size(); vector<vector<int>> dist(V, vector<int>(V)); vector<vector<int>> next(V, vector<int>(V, -1)); // Initialize for (int i = 0; i < V; i++) { for (int j = 0; j < V; j++) { dist[i][j] = graph[i][j]; if (graph[i][j] != INF && i != j) { next[i][j] = j; } } } // Floyd-Warshall for (int k = 0; k < V; k++) { for (int i = 0; i < V; i++) { for (int j = 0; j < V; j++) { if (dist[i][k] != INF && dist[k][j] != INF) { if (dist[i][k] + dist[k][j] < dist[i][j]) { dist[i][j] = dist[i][k] + dist[k][j]; next[i][j] = next[i][k]; } } } } } // Reconstruct path auto getPath = [&](int u, int v) -> vector<int> { if (next[u][v] == -1) return {}; vector<int> path = {u}; while (u != v) { u = next[u][v]; path.push_back(u); } return path; }; // Example: path from 0 to 3 vector<int> path = getPath(0, 3); cout << "Path from 0 to 3: "; for (int v : path) cout << v << " "; cout << endl; }

6. Negative Cycle Detection

After running Floyd-Warshall, if dist[i][i] < 0 for any vertex i, there's a negative cycle:

bool hasNegativeCycle(vector<vector<int>>& dist) { int V = dist.size(); for (int i = 0; i < V; i++) { if (dist[i][i] < 0) { return true; // Negative cycle detected } } return false; }

7. Complexity Analysis

Aspect Complexity
Time O(V³)
Space O(V²)

Comparison with Other Algorithms

Algorithm Time Use Case
Floyd-Warshall O(V³) All pairs, dense graphs
Dijkstra (V times) O(V(E log V)) All pairs, no negative edges
Bellman-Ford (V times) O(V²E) All pairs, negative edges OK

Summary

The Floyd-Warshall algorithm:

  1. Finds shortest paths between all pairs of vertices
  2. Works with negative edge weights (but not negative cycles)
  3. Uses dynamic programming with O(V³) time complexity
  4. Can detect negative cycles
  5. Ideal for dense graphs where we need all-pairs shortest paths
