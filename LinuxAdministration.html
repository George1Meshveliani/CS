1. Initial Server Setup

Choosing a Cloud Provider

Popular options include:

  • DigitalOcean: Simple, developer-friendly
  • AWS EC2: Scalable, enterprise-grade
  • Linode: Good performance/price ratio
  • Vultr: Global locations, competitive pricing
  • Hetzner: European provider, excellent value

Initial Server Configuration

Step 1: Create SSH Key Pair

$ ssh-keygen -t ed25519 -C "your_email@example.com" # Follow prompts to save key (default: ~/.ssh/id_ed25519)

This creates a public/private key pair for secure authentication.

Step 2: Connect to Your Server

$ ssh root@your_server_ip # Or with key: ssh -i ~/.ssh/id_ed25519 root@your_server_ip

Step 3: Update System Packages

# Ubuntu/Debian $ apt update && apt upgrade -y # CentOS/RHEL $ yum update -y # Fedora $ dnf update -y

Step 4: Create Non-Root User

$ adduser username $ usermod -aG sudo username # Ubuntu/Debian $ usermod -aG wheel username # CentOS/RHEL $ su - username

Step 5: Configure Firewall

# UFW (Ubuntu) $ ufw allow OpenSSH $ ufw allow 80/tcp # HTTP $ ufw allow 443/tcp # HTTPS $ ufw enable $ ufw status # firewalld (CentOS/RHEL) $ firewall-cmd --permanent --add-service=ssh $ firewall-cmd --permanent --add-service=http $ firewall-cmd --permanent --add-service=https $ firewall-cmd --reload

2. Domain Management

Registering a Domain

Popular domain registrars:

  • Namecheap: User-friendly, good prices
  • Cloudflare: At-cost pricing, excellent DNS
  • Google Domains: Simple interface
  • Name.com: Good for beginners

Domain Configuration Steps

Step 1: Purchase Domain

Choose a domain name and complete registration. You'll receive access to the domain management panel.

Step 2: Get Your Server IP Address

$ curl ifconfig.me # Or $ hostname -I

This returns your server's public IP address (e.g., 192.0.2.1).

Step 3: Configure DNS Records

In your domain registrar's DNS management panel, add:

  • A Record: @ → your_server_ip (points domain to server)
  • A Record: www → your_server_ip (points www subdomain)
  • CNAME Record: * → yourdomain.com (wildcard subdomain)

3. DNS Configuration

Common DNS Record Types

Record Type Purpose Example
A Maps domain to IPv4 address @ → 192.0.2.1
AAAA Maps domain to IPv6 address @ → 2001:db8::1
CNAME Alias to another domain www → example.com
MX Mail exchange server @ → mail.example.com
TXT Text records (verification, SPF, etc.) SPF, DKIM records

Example DNS Configuration

# Check DNS records $ dig example.com $ nslookup example.com $ host example.com # Test DNS propagation $ dig @8.8.8.8 example.com

4. Web Server Setup

Nginx Installation and Configuration

Install Nginx

# Ubuntu/Debian $ apt install nginx -y # CentOS/RHEL $ yum install nginx -y # Start and enable $ systemctl start nginx $ systemctl enable nginx $ systemctl status nginx

Configure Nginx for Your Domain

$ nano /etc/nginx/sites-available/example.com # Add this configuration: server { listen 80; server_name example.com www.example.com; root /var/www/example.com/html; index index.html index.htm index.nginx-debian.html; location / { try_files $uri $uri/ =404; } } # Create directory and test page $ mkdir -p /var/www/example.com/html $ echo "<h1>Welcome to example.com</h1>" > /var/www/example.com/html/index.html $ chown -R www-data:www-data /var/www/example.com/html # Enable site $ ln -s /etc/nginx/sites-available/example.com /etc/nginx/sites-enabled/ $ nginx -t # Test configuration $ systemctl reload nginx

Apache Installation (Alternative)

Install Apache

# Ubuntu/Debian $ apt install apache2 -y # CentOS/RHEL $ yum install httpd -y # Start and enable $ systemctl start apache2 # or httpd $ systemctl enable apache2

5. SSL Certificates (HTTPS)

Using Let's Encrypt (Free SSL)

Install Certbot

# Ubuntu/Debian $ apt install certbot python3-certbot-nginx -y # CentOS/RHEL $ yum install certbot python3-certbot-nginx -y

Obtain SSL Certificate

# For Nginx $ certbot --nginx -d example.com -d www.example.com # For Apache $ certbot --apache -d example.com -d www.example.com # Auto-renewal test $ certbot renew --dry-run

Certbot automatically configures your web server and sets up auto-renewal.

6. Security Hardening

Essential Security Practices

1. Disable Root Login via SSH

$ nano /etc/ssh/sshd_config # Set these values: PermitRootLogin no PasswordAuthentication no # Use keys only PubkeyAuthentication yes $ systemctl restart sshd

2. Set Up Fail2Ban

$ apt install fail2ban -y $ systemctl enable fail2ban $ systemctl start fail2ban # Check status $ fail2ban-client status sshd

Fail2Ban automatically bans IPs after failed login attempts.

3. Keep System Updated

# Set up automatic security updates $ apt install unattended-upgrades -y $ dpkg-reconfigure -plow unattended-upgrades

4. Configure Regular Backups

# Install backup tools $ apt install rsync -y # Example backup script $ crontab -e # Add: 0 2 * * * rsync -avz /var/www/ /backup/www/

7. Server Monitoring

Basic Monitoring Commands

System Resources

# CPU and Memory $ top $ htop # Install: apt install htop $ free -h $ df -h # Disk usage # Network $ netstat -tulpn $ ss -tulpn $ iptables -L -n # Firewall rules

Log Monitoring

# System logs $ journalctl -xe $ tail -f /var/log/syslog # Nginx logs $ tail -f /var/log/nginx/access.log $ tail -f /var/log/nginx/error.log # Apache logs $ tail -f /var/log/apache2/access.log $ tail -f /var/log/apache2/error.log

Advanced Monitoring Tools

  • Prometheus + Grafana: Advanced metrics and visualization
  • Netdata: Real-time system monitoring
  • Uptime Robot: External uptime monitoring
  • CloudWatch (AWS): Cloud provider monitoring
