Karatsuba Algorithm: Divide and Conquer Multiplication

The Karatsuba algorithm is a fast multiplication algorithm discovered by Anatoly Karatsuba in 1960. It uses a divide-and-conquer approach to multiply two n-digit numbers in O(nlog₂3) ≈ O(n1.585) time, which is faster than the traditional O(n²) schoolbook method.

1. The Problem

Given two n-digit numbers, multiply them efficiently. For very large numbers (like in cryptography), traditional multiplication becomes too slow.

3. Karatsuba Algorithm

The Key Insight

Instead of computing 4 multiplications, Karatsuba computes only 3 multiplications by cleverly combining results.

Mathematical Foundation

Given two n-digit numbers x and y, split them into halves:

x = a × 10n/2 + b

y = c × 10n/2 + d

Where a, b, c, d are n/2-digit numbers

Traditional Multiplication

x × y = (a × 10n/2 + b) × (c × 10n/2 + d)

= ac × 10n + (ad + bc) × 10n/2 + bd

This requires 4 multiplications: ac, ad, bc, bd

Karatsuba's Trick

Notice that: (a + b)(c + d) = ac + ad + bc + bd

Therefore: ad + bc = (a + b)(c + d) - ac - bd

Karatsuba Formula:

x × y = ac × 10n + [(a + b)(c + d) - ac - bd] × 10n/2 + bd

This requires only 3 multiplications:

  1. ac
  2. bd
  3. (a + b)(c + d)

#include <iostream> #include <string> #include <algorithm> using namespace std; // Helper function to add two numbers as strings string addStrings(string num1, string num2) { int i = num1.length() - 1; int j = num2.length() - 1; int carry = 0; string result = ""; while (i >= 0 || j >= 0 || carry > 0) { int digit1 = (i >= 0) ? (num1[i] - '0') : 0; int digit2 = (j >= 0) ? (num2[j] - '0') : 0; int sum = digit1 + digit2 + carry; result = char(sum % 10 + '0') + result; carry = sum / 10; i--; j--; } return result; } // Helper function to subtract two numbers as strings (assumes num1 >= num2) string subtractStrings(string num1, string num2) { int i = num1.length() - 1; int j = num2.length() - 1; int borrow = 0; string result = ""; while (i >= 0) { int digit1 = (num1[i] - '0') - borrow; int digit2 = (j >= 0) ? (num2[j] - '0') : 0; if (digit1 < digit2) { digit1 += 10; borrow = 1; } else { borrow = 0; } result = char(digit1 - digit2 + '0') + result; i--; j--; } // Remove leading zeros while (result.length() > 1 && result[0] == '0') { result = result.substr(1); } return result; } // Helper function to multiply by power of 10 string multiplyByPowerOf10(string num, int power) { return num + string(power, '0'); } // Karatsuba multiplication string karatsuba(string x, string y) { // Make both strings same length int n = max(x.length(), y.length()); while (x.length() < n) x = "0" + x; while (y.length() < n) y = "0" + y; // Base case: if numbers are small, use direct multiplication if (n == 1) { int product = (x[0] - '0') * (y[0] - '0'); return to_string(product); } // Split into halves int mid = n / 2; string a = x.substr(0, n - mid); string b = x.substr(n - mid); string c = y.substr(0, n - mid); string d = y.substr(n - mid); // Recursive calls: 3 multiplications string z0 = karatsuba(a, c); // ac string z2 = karatsuba(b, d); // bd string z1 = karatsuba(addStrings(a, b), addStrings(c, d)); // (a+b)(c+d) // Calculate z1 - z0 - z2 = ad + bc string ad_plus_bc = subtractStrings(subtractStrings(z1, z0), z2); // Result: z0 * 10^(2*mid) + (ad+bc) * 10^mid + z2 string term1 = multiplyByPowerOf10(z0, 2 * mid); string term2 = multiplyByPowerOf10(ad_plus_bc, mid); string result = addStrings(addStrings(term1, term2), z2); // Remove leading zeros while (result.length() > 1 && result[0] == '0') { result = result.substr(1); } return result; } int main() { string x = "1234"; string y = "5678"; cout << "Multiplying: " << x << " × " << y << endl; string result = karatsuba(x, y); cout << "Result: " << result << endl; cout << "Expected: " << (1234LL * 5678LL) << endl; // Large number example x = "3141592653589793238462643383279502884197169399375105820974944592"; y = "2718281828459045235360287471352662497757247093699959574966967627"; cout << "\nLarge number multiplication:" << endl; result = karatsuba(x, y); cout << "Result length: " << result.length() << " digits" << endl; cout << "First 50 digits: " << result.substr(0, 50) << "..." << endl; return 0; }

Simplified Integer Version (for smaller numbers)

#include <iostream> #include <cmath> using namespace std; long long karatsuba(long long x, long long y) { // Base case if (x < 10 || y < 10) { return x * y; } // Calculate number of digits int n = max((int)log10(x) + 1, (int)log10(y) + 1); int m = n / 2; long long power = pow(10, m); // Split numbers long long a = x / power; long long b = x % power; long long c = y / power; long long d = y % power; // Three recursive multiplications long long z0 = karatsuba(a, c); long long z2 = karatsuba(b, d); long long z1 = karatsuba(a + b, c + d); // Result return z0 * power * power + (z1 - z0 - z2) * power + z2; } int main() { long long x = 1234; long long y = 5678; cout << "Karatsuba: " << karatsuba(x, y) << endl; cout << "Direct: " << x * y << endl; return 0; }

6. Complexity Analysis

Time Complexity

Recurrence Relation: T(n) = 3T(n/2) + O(n)

Where:

  • 3T(n/2): Three recursive calls on half-sized numbers
  • O(n): Time for additions, subtractions, and shifts

Solution using Master Theorem:

T(n) = O(nlog₂3) ≈ O(n1.585)

Space Complexity

O(log n) - Depth of recursion tree

Each recursive call uses O(n) space for string operations, but space is reused.

7. Comparison with Other Methods

When to Use Karatsuba

  • Advantages: Faster than schoolbook for large numbers, relatively simple to implement
  • Disadvantages: Overhead for small numbers, not as fast as FFT-based methods for very large numbers
  • Best for: Numbers with 100-10,000 digits

Real-World Applications

  • Cryptography: RSA encryption, large prime number generation
  • Arbitrary Precision Arithmetic: Mathematical libraries, scientific computing
  • Computer Algebra Systems: Symbolic computation, polynomial multiplication

Summary

The Karatsuba algorithm demonstrates the power of divide-and-conquer:

  1. Reduces 4 multiplications to 3 using algebraic manipulation
  2. Achieves O(n1.585) time complexity vs O(n²) for schoolbook
  3. Uses recursion to handle large numbers efficiently
  4. Essential for cryptographic and high-precision arithmetic applications
