2) Time and Memory Computational Complexity

Complexity describes growth of resource usage as input size n increases.

AlgorithmBestAverageWorstAux Memory
QuicksortO(n log n)O(n log n)O(n²)O(log n) stack avg, O(n) worst
QuickhullO(n log n)~O(n log n)O(n²)O(n) typical recursion/data

Important: Big-O hides constants. For real systems, cache locality, branch prediction, and memory allocation overhead can dominate smaller inputs.

3) How to Bound Computational Complexity (Recurrences)

Divide-and-conquer algorithms are naturally described by recurrence equations.

Quicksort recurrence: T(n) = T(k) + T(n-k-1) + Theta(n) Balanced case (k ≈ n/2): T(n) = 2T(n/2) + Theta(n) => Theta(n log n) Worst case (k = 0 or n-1): T(n) = T(n-1) + Theta(n) => Theta(n^2)

For balanced splits, Master theorem applies directly:

\[ T(n) = aT\left(\frac{n}{b}\right) + f(n) \]

If a=2, b=2, and f(n)=Theta(n), then T(n)=Theta(n\log n).

4) Partition Procedure and Quicksort Algorithm

Lomuto Partition (simple, educational)

#include <bits/stdc++.h> using namespace std; int partitionLomuto(vector<int>& a, int lo, int hi) { int pivot = a[hi]; int i = lo - 1; for (int j = lo; j < hi; ++j) { if (a[j] <= pivot) { ++i; swap(a[i], a[j]); } } swap(a[i + 1], a[hi]); return i + 1; } void quicksort(vector<int>& a, int lo, int hi) { if (lo >= hi) return; int p = partitionLomuto(a, lo, hi); quicksort(a, lo, p - 1); quicksort(a, p + 1, hi); }

Optimization ideas

  • Randomized pivot to avoid adversarial worst-case
  • Median-of-three pivot selection
  • Switch to insertion sort for tiny partitions
  • Tail-recursion elimination to reduce stack depth

5) Implementations: Haskell vs C/C++

Haskell (declarative, immutable lists)

quicksort :: Ord a => [a] -> [a] quicksort [] = [] quicksort (x:xs) = quicksort [y | y <- xs, y <= x] ++ [x] ++ quicksort [y | y <- xs, y > x]

C++ (in-place, imperative)

vector<int> a = {9,4,7,3,1,8,2,5,6}; quicksort(a, 0, (int)a.size() - 1);

6) Quickhull Algorithm for Convex Hull

Quickhull finds convex hull points by recursively selecting the point farthest from a line segment and splitting into subproblems, conceptually similar to Quicksort partitioning.

Core idea

  1. Find leftmost and rightmost points: A, B
  2. Split remaining points into two sets by line AB
  3. Pick farthest point P from line segment
  4. Recurse on A-P and P-B subsets

C++ Skeleton

struct Point { double x, y; }; double cross(const Point& a, const Point& b, const Point& c) { return (b.x - a.x) * (c.y - a.y) - (b.y - a.y) * (c.x - a.x); } void quickHullRec(const vector<Point>& pts, Point a, Point b, int side, vector<Point>& hull) { int idx = -1; double maxDist = 0.0; for (int i = 0; i < (int)pts.size(); ++i) { double val = cross(a, b, pts[i]) * side; if (val > 0 && abs(val) > maxDist) { idx = i; maxDist = abs(val); } } if (idx == -1) { hull.push_back(a); hull.push_back(b); return; } Point p = pts[idx]; quickHullRec(pts, a, p, -((cross(a, p, b) > 0) ? 1 : -1), hull); quickHullRec(pts, p, b, -((cross(p, b, a) > 0) ? 1 : -1), hull); }

Result: Quickhull is very fast in many practical geometric datasets, but still has quadratic worst-case behavior on adversarial or highly pathological point distributions.

Medium

Problem M1: Randomized Quicksort Medium

Task: Improve quicksort by selecting a random pivot before partitioning.

int partitionRandom(vector<int>& a, int lo, int hi) { int pivotPos = lo + rand() % (hi - lo + 1); swap(a[pivotPos], a[hi]); return partitionLomuto(a, lo, hi); } void quicksortRandom(vector<int>& a, int lo, int hi) { if (lo >= hi) return; int p = partitionRandom(a, lo, hi); quicksortRandom(a, lo, p - 1); quicksortRandom(a, p + 1, hi); }

Problem M2: Solve the Recurrence Medium

Task: Bound complexity for T(n) = 2T(n/2) + n and T(n) = T(n-1) + n.

1) T(n) = 2T(n/2) + n Master theorem: a=2, b=2, f(n)=n n^(log_b a)=n^1=n, so case 2: T(n)=Theta(n log n) 2) T(n) = T(n-1) + n Unroll: T(n)=T(1)+2+3+...+n => Theta(n^2)

Hard

Problem H1: Iterative Quicksort Hard

Task: Implement quicksort without recursion using an explicit stack of ranges.

void quicksortIterative(vector<int>& a) { if (a.empty()) return; vector<pair<int,int>> st; st.push_back({0, (int)a.size() - 1}); while (!st.empty()) { auto [lo, hi] = st.back(); st.pop_back(); if (lo >= hi) continue; int p = partitionLomuto(a, lo, hi); if (p - 1 > lo) st.push_back({lo, p - 1}); if (p + 1 < hi) st.push_back({p + 1, hi}); } }
