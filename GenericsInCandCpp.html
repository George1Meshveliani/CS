1) What Generic Programming Means

Generic programming means writing one algorithm or data structure that can work with multiple types, instead of rewriting the same logic for int, double, string, and so on.

Common goals:

  • Code reuse without copy-paste
  • Strong correctness guarantees
  • High performance and low overhead
  • Clear, scalable library design

2) Generics in C

C has no templates, so generics are implemented through language patterns:

2a) Macros

#define MAX(a, b) ((a) > (b) ? (a) : (b)) int main() { int a = MAX(3, 10); double b = MAX(3.5, 2.1); }
Important: macros are text substitution. They are not type-safe and can evaluate expressions more than once (e.g. MAX(i++, j++)).

2b) void* + callbacks (library style)

This is the style used by qsort: data pointer + element size + comparator function.

#include <stdlib.h> int cmpInt(const void* a, const void* b) { int x = *(const int*)a; int y = *(const int*)b; return (x > y) - (x < y); } int main() { int arr[] = {7, 2, 9, 1, 5}; qsort(arr, 5, sizeof(int), cmpInt); }

Pros: very flexible and portable. Cons: manual casting and weaker compile-time checks.

2c) C11 _Generic

_Generic provides compile-time selection by expression type.

#include <stdio.h> #define type_name(x) _Generic((x), \ int: "int", \ double: "double", \ char*: "char*", \ default: "other") int main() { printf("%s\n", type_name(10)); // int printf("%s\n", type_name(3.14)); // double }

3) Generics in C++

C++ uses templates to generate type-specific code during compilation.

3a) Function templates

template <typename T> T my_max(const T& a, const T& b) { return (a < b) ? b : a; } int main() { int x = my_max(4, 9); double y = my_max(1.2, 3.4); }

3b) Class templates

#include <vector> using namespace std; template <typename T> class Stack { vector<T> data; public: void push(const T& v) { data.push_back(v); } void pop() { if (!data.empty()) data.pop_back(); } const T& top() const { return data.back(); } bool empty() const { return data.empty(); } };

3c) Template constraints (C++20 concepts)

#include <concepts> template <typename T> concept Number = std::integral<T> || std::floating_point<T>; template <Number T> T add(T a, T b) { return a + b; }
Concepts improve readability and compiler diagnostics by expressing requirements directly in the function signature.

4) Core Principles

  • Abstraction: separate algorithm behavior from concrete type.
  • Type requirements: define what operations the type must support.
  • Zero-overhead intent: abstractions should not force runtime penalties.
  • Composability: generic components should work together predictably.
  • Safety: catch misuse early (prefer compile-time checks).

5) Similarities and Differences

AspectCC++
Main mechanismMacros, void*, _GenericTemplates, concepts
Type safetyMostly manualStrong compile-time type checking
Error timingOften runtime if casts are wrongMostly compile-time
Performance modelCan be fast but manual and error-proneUsually zero-cost abstractions after optimization
Library ecosystemConvention-driven APIsTemplate-heavy STL and generic algorithms

Key similarities

  • Both languages support reusable, type-agnostic design.
  • Both can produce highly efficient low-level code.
  • Both benefit from clear API contracts.

Key differences

  • C++ has built-in generic mechanisms; C simulates generics with idioms.
  • C++ catches more mistakes at compile time.
  • C generic interfaces are often simpler but require more discipline.

6) Practical Guidance

Use C style generics when writing lightweight C libraries, systems APIs, or strict C-only projects.

Use C++ templates/concepts when building reusable containers/algorithms with strong type safety and maintainability.

Rule of thumb: If your project is in C++, prefer templates first. If your project is in pure C, use void*-based APIs carefully and document type contracts clearly.
